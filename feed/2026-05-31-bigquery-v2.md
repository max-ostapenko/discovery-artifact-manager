# Iceberg support for timestamp precision and UDF tweaks

**Date:** 2026-05-31  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery expands timestamp precision autodetection to Iceberg External Tables and clarifies Python UDF concurrency settings.

## Details

The timestampTargetPrecision field within ExternalDataConfiguration and JobConfigurationLoad now explicitly supports the Iceberg External Table format. This allows developers to specify subsecond precision (e.g., 6 or 12 digits) during autodetection for Iceberg data, similar to existing support for CSV, Parquet, and Avro. Additionally, the description for containerRequestConcurrency in ExternalRuntimeOptions was updated to clarify that it specifically controls concurrency for Python UDF container instances.
