# Table-valued arguments and multi-cloud storage stats

**Date:** 2026-07-09  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

BigQuery now supports table-valued arguments for routines and provides granular object storage statistics for multi-cloud queries. Developers can also now force on-demand billing by setting reservations to 'none'.

## Details

The Argument schema has been updated with FIXED_TABLE and ANY_TABLE types, enabling routines to accept structured table inputs via the new tableType field. For multi-cloud and Lakehouse users, the new ObjectStorageStats resource provides detailed metrics on cacheBytesRead and objectStorageBytesRead across GCP, AWS, and Azure. Additionally, JobStatistics now includes fields for tracking global query regions and parent jobs, while the reservation field in job configurations now supports a 'none' value to explicitly trigger on-demand billing, provided the reservation_override_mode is correctly configured.
