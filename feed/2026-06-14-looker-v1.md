# Looker Core adds Soft Delete recovery and Release Channels

**Date:** 2026-06-14  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides control over update frequency with release channel settings.

## Details

The API introduces a comprehensive soft-delete lifecycle. Developers can now use the `instances.undelete` method to recover instances, and the `instances.list` method now accepts a `showDeleted` parameter. New schema fields like `softDeleteReason` (billing issues, trial expiry, or customer request) and `suspendedTime` provide visibility into deactivated instances. Additionally, the `releaseChannel` field allows selection between RAPID, REGULAR, and STABLE update tracks, while `acceleratedSecurityPatchEnabled` offers a new toggle for faster security updates.
