# Looker Core adds Soft Delete and Release Channels

**Date:** 2026-06-17  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides granular control over update cycles with release channel settings.

## Details

This update introduces a 'soft delete' lifecycle for Looker instances. Developers can now use the new `instances.undelete` method to recover instances, and the `instances.list` method now accepts a `showDeleted` parameter to view them. The `Instance` schema has been expanded with `softDeleteReason` and `suspendedTime` to provide context on why an instance was removed (e.g., billing issues or trial expiration). Additionally, a new `releaseChannel` field allows users to choose between RAPID, REGULAR, and STABLE update tracks, while `acceleratedSecurityPatchEnabled` offers more control over security maintenance.
