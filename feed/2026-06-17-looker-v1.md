# Looker Core adds Instance Recovery and Release Channels

**Date:** 2026-06-17  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports soft-delete recovery and granular control over update cadences with new release channel settings.

## Details

This update introduces a new `undelete` method to the instances resource, allowing developers to restore instances that were soft-deleted due to billing issues, trial expirations, or manual requests. To support this, the `list` method now accepts a `showDeleted` parameter, and the Instance schema includes new fields like `softDeleteReason` and `suspendedTime`. Additionally, developers can now configure the `releaseChannel` (RAPID, REGULAR, STABLE) and toggle `acceleratedSecurityPatchEnabled` to manage how and when their instances receive updates.
