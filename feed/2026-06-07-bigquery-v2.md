# Iceberg support for timestamp precision autodetection

**Date:** 2026-06-07  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded timestamp precision autodetection to include Iceberg External Tables and clarified concurrency settings for Python UDFs.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` schemas now explicitly supports Iceberg External Tables. This allows developers to control sub-second precision (e.g., 6 or 12 digits) during schema autodetection for Iceberg sources. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to specify it applies to Python UDF container instances, providing better clarity for developers tuning user-defined function performance.
