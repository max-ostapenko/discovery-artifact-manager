# Iceberg support for timestamp precision autodetection

**Date:** 2026-06-07  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded timestamp precision autodetection to include Iceberg External Tables and clarified Python UDF concurrency settings.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly lists 'Iceberg External Table' as a supported format for timestamp autodetection. This allows developers to control subsecond precision (e.g., 6 or 12 digits) when loading or querying Iceberg data. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to clarify that it specifically governs Python UDF container instances.
