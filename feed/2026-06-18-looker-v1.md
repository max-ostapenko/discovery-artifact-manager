# Looker v1 adds Soft Delete recovery and Release Channels

**Date:** 2026-06-18  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and provides more granular control over update cycles with release channel settings.

## Details

The API introduces a significant safety feature with the new `instances.undelete` method and a `showDeleted` parameter for the `list` method, allowing developers to recover instances lost to accidental deletion, billing issues, or trial expirations. Additionally, the `Instance` schema now supports `releaseChannel` (RAPID, REGULAR, STABLE) and `acceleratedSecurityPatchEnabled`, giving admins better control over their environment's maintenance and security posture. New output-only fields like `softDeleteReason` and `suspendedTime` provide better visibility into the lifecycle state of an instance.
