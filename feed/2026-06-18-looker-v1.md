# Looker v1 adds Soft Delete recovery and Release Channels

**Date:** 2026-06-18  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and provides granular control over update frequency with Release Channels.

## Details

The instances.list method now includes a `showDeleted` flag, paired with a new `undelete` endpoint to restore instances. The Instance resource gained `releaseChannel` (Rapid, Regular, Stable) and `acceleratedSecurityPatchEnabled` fields, allowing teams to manage their update risk profile. New output-only fields like `softDeleteReason` and `suspendedTime` provide better visibility into instance lifecycle events, specifically identifying if a deletion was caused by billing issues, trial expiration, or manual requests.
