# Looker API adds Instance Undelete and Release Channels

**Date:** 2026-06-18  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides visibility into soft-deleted instances. Developers can also now programmatically manage update cadences using release channels.

## Details

The API introduces a new `undelete` method for Looker instances, complemented by a `showDeleted` parameter in the `list` method to help manage lifecycle recovery. New schema fields include `softDeleteReason` (tracking if an instance was removed due to billing, trial expiry, or user request), `suspendedTime`, and `releaseChannel` which allows selecting between RAPID, REGULAR, and STABLE update tracks. Additionally, a new `acceleratedSecurityPatchEnabled` boolean has been added to the Instance resource.
