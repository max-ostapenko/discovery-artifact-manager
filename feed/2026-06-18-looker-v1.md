# Looker v1 adds Instance Recovery and Release Channels

**Date:** 2026-06-18  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker developers can now recover soft-deleted instances and control update frequency via new release channels.

## Details

The API introduces a new `projects.locations.instances.undelete` method and a `showDeleted` query parameter for the `list` method, enabling a full soft-delete recovery workflow. The `Instance` schema has been expanded with `releaseChannel` (supporting RAPID, REGULAR, and STABLE cadences) and `acceleratedSecurityPatchEnabled` for faster security rollouts. New output-only fields like `softDeleteReason` and `suspendedTime` provide better visibility into why an instance was taken offline, whether due to billing issues, trial expiration, or customer request.
