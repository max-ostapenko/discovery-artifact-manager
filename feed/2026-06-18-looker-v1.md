# Looker Core adds Instance Undelete and Release Channels

**Date:** 2026-06-18  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports soft-deletion recovery and configurable release channels for instances, providing better control over lifecycle and updates.

## Details

A new `instances.undelete` method has been added, allowing developers to recover instances that were previously deleted. To support this, the `instances.list` method now accepts a `showDeleted` parameter. Additionally, the `Instance` schema has been expanded with `releaseChannel` (RAPID, REGULAR, STABLE) to manage update frequency, and `acceleratedSecurityPatchEnabled` for opting into faster security rollouts. New output-only fields like `softDeleteReason` and `suspendedTime` provide metadata on why an instance was taken offline, such as billing issues or trial expirations.
