# Looker Core adds Soft-Delete Recovery and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides granular control over update cycles with release channel settings.

## Details

The API introduces a soft-delete lifecycle for Looker instances. Developers can now list deleted instances using the new `showDeleted` query parameter on the list method and restore them using the `instances.undelete` POST method. The `Instance` schema has been expanded with `softDeleteReason` (identifying if suspension was due to billing, trial expiry, or user request) and `suspendedTime`. Furthermore, new configuration options allow for selecting a `releaseChannel` (RAPID, REGULAR, STABLE) and toggling `acceleratedSecurityPatchEnabled` to manage how and when instances receive updates.
