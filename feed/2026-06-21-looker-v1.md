# Looker adds Instance Undelete and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker instances now support a soft-delete lifecycle with recovery options and configurable release channels for updates.

## Details

A new `instances.undelete` method has been introduced, allowing developers to recover instances that were soft-deleted due to billing issues, trial expirations, or manual requests. To support this, the `instances.list` method now includes a `showDeleted` parameter, and the Instance schema adds `softDeleteReason` and `suspendedTime` fields. Additionally, developers can now programmatically manage update cadences via the new `releaseChannel` field (supporting RAPID, REGULAR, and STABLE) and toggle `acceleratedSecurityPatchEnabled` for faster security updates.
