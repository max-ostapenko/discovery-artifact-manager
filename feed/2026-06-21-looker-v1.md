# Looker adds Soft Delete recovery and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker instances now support a soft-delete lifecycle, allowing for recovery via a new undelete method, alongside new controls for update release channels.

## Details

The API introduces a significant safety net with the new `instances.undelete` method and a `showDeleted` parameter for `instances.list`. Instances in a soft-deleted state now expose `softDeleteReason` (e.g., billing issues or trial expiration) and `suspendedTime`. Furthermore, developers can now programmatically manage update cadences using the new `releaseChannel` field (RAPID, REGULAR, STABLE) and toggle `acceleratedSecurityPatchEnabled` for faster security updates.
