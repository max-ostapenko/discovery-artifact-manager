# Looker Core adds Soft Delete recovery and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides granular control over update cycles with release channels.

## Details

This update introduces a 'soft delete' lifecycle for Looker instances. A new `instances.undelete` method allows for the recovery of instances, while the `instances.list` method now accepts a `showDeleted` parameter. The `Instance` schema has been expanded with `softDeleteReason` and `suspendedTime` to provide context on why an instance was removed. Additionally, developers can now manage update cadences using the new `releaseChannel` field (supporting RAPID, REGULAR, and STABLE) and toggle `acceleratedSecurityPatchEnabled` for faster security updates.
