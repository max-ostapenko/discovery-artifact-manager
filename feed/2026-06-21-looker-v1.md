# Looker adds Instance Undelete and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core instances now support a soft-delete lifecycle with a new recovery method and configurable release channels for updates.

## Details

The API introduces a significant lifecycle update with the new `instances.undelete` method, allowing developers to recover instances deleted due to billing issues, trial expirations, or manual requests. To support this, `instances.list` now accepts a `showDeleted` parameter, and the Instance schema includes `softDeleteReason` and `suspendedTime`. Additionally, developers can now specify a `releaseChannel` (RAPID, REGULAR, or STABLE) to control update frequency and toggle `acceleratedSecurityPatchEnabled` for faster security maintenance.
