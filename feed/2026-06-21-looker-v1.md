# Looker v1 adds Instance Undelete and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker developers can now recover soft-deleted instances and control update cadences via new release channels and security patch settings.

## Details

This update introduces a major lifecycle management feature: the `undelete` method for Looker instances. To support this, the `list` method now accepts a `showDeleted` boolean parameter, and the `Instance` schema includes new output-only fields `softDeleteReason` and `suspendedTime`. Additionally, developers can now manage update frequency using the `releaseChannel` field (supporting RAPID, REGULAR, and STABLE) and opt into `acceleratedSecurityPatchEnabled` for faster security updates.
