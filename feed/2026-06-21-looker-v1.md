# Looker Core adds Soft-Delete Recovery and Release Channels

**Date:** 2026-06-21  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides more control over update cadences with releaseChannel settings.

## Details

The API now includes a formal soft-delete lifecycle. Developers can use the new `instances.undelete` method to recover instances, and the `instances.list` method now accepts a `showDeleted` parameter. The `Instance` schema has been expanded with `softDeleteReason` and `suspendedTime` to provide context on why an instance was removed. Additionally, two new configuration options are available: `releaseChannel` (allowing selection between RAPID, REGULAR, and STABLE updates) and `acceleratedSecurityPatchEnabled` for faster security rollouts.
