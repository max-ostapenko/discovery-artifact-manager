# Iceberg support added to timestamp autodetection

**Date:** 2026-06-06  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded its timestamp autodetection precision settings to include Iceberg External Tables and clarified Python UDF concurrency options.

## Details

The `timestampTargetPrecision` field within `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly supports Iceberg External Tables. This allows developers to control how TIMESTAMP types are handled during autodetection for Iceberg sources (e.g., choosing between 6 or 12 digits of subsecond precision). Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to specify it applies to Python UDF container instances rather than generic Cloud Run instances.
