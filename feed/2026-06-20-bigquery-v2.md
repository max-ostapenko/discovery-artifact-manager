# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-20  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has updated its discovery document to include specific IAM permission requirements for nearly every method, simplifying the process of configuring least-privilege access.

## Details

The update adds explicit IAM permission requirements to the descriptions of core resources including datasets, jobs, tables, routines, and rowAccessPolicies. For instance, the documentation for tabledata.insertAll now clarifies that it requires bigquery.tables.updateData, bigquery.tables.get, and bigquery.datasets.get. Additionally, ExternalDataConfiguration and JobConfigurationLoad now mention support for Iceberg External Tables regarding timestampTargetPrecision, and ExternalRuntimeOptions clarifies the use of Python UDF container instances.
