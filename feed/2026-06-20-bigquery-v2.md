# BigQuery API adds explicit IAM requirements to method docs

**Date:** 2026-06-20  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

The BigQuery v2 discovery document has been overhauled to explicitly list the IAM permissions required for nearly every API method, significantly simplifying security troubleshooting.

## Details

Descriptions for resources including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now feature a dedicated '# IAM Permissions' section. For instance, the `tabledata.insertAll` method now explicitly documents its requirement for `bigquery.tables.updateData`, `bigquery.tables.get`, and `bigquery.datasets.get`. Beyond IAM updates, `timestampTargetPrecision` now supports Iceberg External Tables, and `ExternalRuntimeOptions` has been clarified to refer specifically to Python UDF container instances rather than generic Cloud Run instances.
