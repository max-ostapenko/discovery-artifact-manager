# BigQuery API clarifies IAM requirements across all methods

**Date:** 2026-06-20  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to explicitly list required IAM permissions for nearly every resource method, providing much-needed clarity for security and access troubleshooting.

## Details

This update adds explicit IAM permission requirements to the descriptions of methods across datasets, jobs, models, routines, rowAccessPolicies, and tables. For example, it clarifies that `jobs.get` requires `bigquery.jobs.get` on the resource, but the job creator only needs `bigquery.jobs.create` on the project. It also notes that `jobs.list` results are filtered or redacted based on whether the caller has `bigquery.jobs.listAll` or `bigquery.jobs.list`. Minor schema description updates also confirm that `timestampTargetPrecision` now supports Iceberg External Tables and clarify that `ExternalRuntimeOptions` applies to Python UDF container instances.
