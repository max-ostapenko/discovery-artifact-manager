# Explicit IAM requirements added to BigQuery API docs

**Date:** 2026-06-20  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to explicitly list the IAM permissions required for nearly every method, providing much-needed clarity for security auditing and troubleshooting.

## Details

Almost every major resource—including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables—now features a dedicated '# IAM Permissions' section in its method descriptions. For instance, jobs.getQueryResults now explicitly documents the requirement for bigquery.jobs.get and bigquery.tables.getData. Beyond IAM, the ExternalDataConfiguration schema now includes Iceberg External Tables in its timestampTargetPrecision support, and ExternalRuntimeOptions clarifies that containerRequestConcurrency applies specifically to Python UDF container instances.
