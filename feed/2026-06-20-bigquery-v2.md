# BigQuery API clarifies IAM requirements across all methods

**Date:** 2026-06-20  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

A massive documentation update explicitly defines the IAM permissions required for nearly every BigQuery API method, from dataset management to job execution.

## Details

While no functional schema changes were introduced, the discovery document now serves as a definitive reference for least-privilege configuration. Key clarifications include the specific permissions for jobs.getQueryResults (requiring both job and table-level access), tabledata.insertAll (requiring dataset and table-level access), and the nuances of job ownership permissions. Additionally, timestampTargetPrecision now explicitly mentions support for Iceberg External Tables, and transferredBytes statistics are clarified for BigQuery Omni use cases.
