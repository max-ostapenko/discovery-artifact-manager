# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, simplifying access troubleshooting.

## Details

Descriptions for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now feature a dedicated '# IAM Permissions' section. For example, jobs.getQueryResults now explicitly notes the need for bigquery.jobs.get and bigquery.tables.getData, while rowAccessPolicies methods detail the specific combinations of create, delete, and setIamPolicy permissions required.
