# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 Discovery Document has been updated to include specific IAM permission requirements for nearly every method, clarifying exactly what access is needed for datasets, jobs, and tables.

## Details

While no functional code changes were introduced, the API descriptions for almost 40 methods—including datasets, jobs, models, routines, and rowAccessPolicies—now feature a dedicated '# IAM Permissions' section. For example, `jobs.query` and `jobs.insert` now explicitly state they require `bigquery.jobs.create` on the project, and `tabledata.insertAll` (streaming) now lists its dependency on `bigquery.tables.updateData`, `bigquery.tables.get`, and `bigquery.datasets.get` permissions.
