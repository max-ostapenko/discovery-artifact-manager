# BigQuery API adds explicit IAM permission documentation

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

The BigQuery discovery document has been updated to explicitly list the IAM permissions required for nearly every API method, from dataset management to job execution.

## Details

While no functional schema changes were introduced, the API descriptions now serve as a definitive reference for IAM requirements. Notable clarifications include the specific permissions needed for job creators versus other users on `jobs.get` and `jobs.cancel`, and the multi-permission requirements for `rowAccessPolicies` and `tabledata.insertAll`. Additionally, documentation for `timestampTargetPrecision` now confirms support for Iceberg External Tables, and `transferredBytes` in job statistics has been clarified for BigQuery Omni use cases.
