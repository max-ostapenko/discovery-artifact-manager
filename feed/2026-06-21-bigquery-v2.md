# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for almost every method. While no functional changes were made, this provides much-needed clarity for developers managing access control and debugging 403 errors.

## Details

Descriptions for core resources including datasets, jobs, models, routines, and tables now explicitly state the required IAM permissions (e.g., bigquery.jobs.create, bigquery.tables.getData). Notable clarifications include the specific permissions needed for job creators versus other users on jobs.get and jobs.cancel. Additionally, timestampTargetPrecision now officially supports Iceberg External Tables, and JobStatistics2.transferredBytes has been redefined to clarify data movement in BigQuery Omni scenarios.
