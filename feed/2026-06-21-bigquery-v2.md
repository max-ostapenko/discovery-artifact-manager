# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 Discovery Document has been updated to include detailed IAM permission requirements for nearly every method, simplifying access control troubleshooting.

## Details

Descriptions for methods across datasets, jobs, tables, models, and routines now feature a dedicated '# IAM Permissions' section. This clarifies exactly which permissions are needed for specific calls; for instance, jobs.getQueryResults now explicitly documents the requirement for bigquery.tables.getData on the destination table. It also clarifies behavior for listing methods, noting that results are filtered based on the caller's existing permissions rather than requiring a global list permission.
