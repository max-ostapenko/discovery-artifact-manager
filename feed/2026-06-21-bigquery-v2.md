# BigQuery API now explicitly documents IAM requirements

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, significantly easing security configuration.

## Details

Descriptions for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now include a dedicated '# IAM Permissions' section. For example, jobs.getQueryResults now explicitly lists the need for bigquery.jobs.get and bigquery.tables.getData. Additionally, timestampTargetPrecision now supports Iceberg External Tables, and transferredBytes in job statistics has been clarified for BigQuery Omni use cases.
