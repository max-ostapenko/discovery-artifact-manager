# Explicit IAM Permission Requirements Added to API Docs

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

BigQuery has updated its API discovery document to explicitly list the IAM permissions required for nearly every method, including datasets, jobs, tables, and row access policies.

## Details

This massive documentation update adds a dedicated '# IAM Permissions' section to the descriptions of core methods. For instance, `jobs.insert` and `jobs.query` now detail the nuances between standard queries and complex DDL/DCL operations. Other methods like `tabledata.insertAll` (streaming) now explicitly list the required triad of `bigquery.tables.updateData`, `bigquery.tables.get`, and `bigquery.datasets.get`. Additionally, `ExternalDataConfiguration` descriptions were updated to include support for Iceberg External Tables.
