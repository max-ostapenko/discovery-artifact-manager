# BigQuery API adds explicit IAM permissions to documentation

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method. This makes it significantly easier to debug 403 Forbidden errors and configure least-privilege service accounts.

## Details

Descriptions for resources including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now feature a dedicated '# IAM Permissions' section. For example, jobs.getQueryResults now explicitly notes it requires bigquery.jobs.get and bigquery.tables.getData, while tabledata.insertAll clarifies the need for bigquery.tables.updateData, bigquery.tables.get, and bigquery.datasets.get. This change covers 39 different methods across the API surface.
