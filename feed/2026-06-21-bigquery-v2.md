# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 API documentation has been overhauled to explicitly list the IAM permissions required for nearly every method, simplifying access control debugging.

## Details

Nearly 40 methods across datasets, jobs, tables, models, routines, and row-level access policies now include a dedicated '# IAM Permissions' section in their descriptions. This clarifies complex requirements, such as the specific permissions needed for `jobs.getQueryResults` (which varies if you are the job creator) and the multi-permission requirements for `tabledata.insertAll` and `rowAccessPolicies` operations. While these are documentation-only changes, they provide critical visibility into the underlying security model for developers.
