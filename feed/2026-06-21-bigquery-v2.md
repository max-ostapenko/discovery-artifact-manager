# BigQuery API adds explicit IAM requirements to method docs

**Date:** 2026-06-21  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

Nearly every method in the BigQuery v2 API has been updated to explicitly document the required IAM permissions directly in the discovery document.

## Details

This documentation overhaul covers datasets, jobs, tables, models, routines, and row access policies. It clarifies nuanced permission logic, such as how job creators only need project-level `bigquery.jobs.create` to manage their own jobs, while others require resource-specific permissions. It also explicitly lists the multi-level permissions required for streaming inserts (`tabledata.insertAll`) and clarifies the filtering behavior of list operations for projects and datasets.
