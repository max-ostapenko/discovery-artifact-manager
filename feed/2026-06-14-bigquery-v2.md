# BigQuery API now explicitly documents IAM requirements

**Date:** 2026-06-14  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

The BigQuery discovery document has been updated to include specific IAM permission requirements for nearly every API method, significantly aiding in debugging and security configuration.

## Details

Method descriptions across datasets, jobs, models, routines, rowAccessPolicies, and tables now include an '# IAM Permissions' section. This clarifies exactly which permissions are needed, such as the distinction for job creators needing `bigquery.jobs.create` versus others needing `bigquery.jobs.get`. Additionally, `timestampTargetPrecision` documentation was updated to include support for Iceberg External Tables, and `ExternalRuntimeOptions` now specifies its use for Python UDF container instances.
