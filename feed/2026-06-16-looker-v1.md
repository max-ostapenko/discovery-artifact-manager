# Looker v1 adds Soft Delete recovery and Release Channels

**Date:** 2026-06-16  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and provides granular control over update cycles with releaseChannel settings.

## Details

The instances resource gains a new `undelete` method and a `showDeleted` filter for the `list` method, allowing developers to recover instances deleted by customers, billing issues, or trial expirations. New schema fields like `softDeleteReason` and `suspendedTime` provide metadata on these states. Additionally, the API now supports selecting a `releaseChannel` (RAPID, REGULAR, STABLE) and toggling `acceleratedSecurityPatchEnabled` for more precise environment management.
