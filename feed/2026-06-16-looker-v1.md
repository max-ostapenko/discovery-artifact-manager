# Looker v1: Soft-delete recovery and release channels

**Date:** 2026-06-16  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and soft-delete tracking. Developers can also now configure release channels and accelerated security patches.

## Details

The instances.list method gains a `showDeleted` boolean parameter to view instances in a soft-deleted state. A new `instances.undelete` POST method allows for restoration of these instances. Additionally, the Instance schema is expanded with `releaseChannel` (RAPID, REGULAR, STABLE) for update cadence control, `softDeleteReason` to diagnose why an instance was suspended (e.g., billing issues or trial expiration), and `acceleratedSecurityPatchEnabled` for faster security rollouts.
