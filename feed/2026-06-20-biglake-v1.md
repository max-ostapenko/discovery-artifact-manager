# BigLake API Rebrands to Lakehouse API

**Date:** 2026-06-20  
**API:** `biglake.v1`  
**Impact:** Low  

## Summary

The BigLake API has been rebranded to the Lakehouse API, signaling a broader focus on multi-engine metadata sharing and cross-cloud federation.

## Details

This update is purely descriptive but significant for product positioning. The BigLake API is now officially referred to as the Lakehouse API. The updated documentation highlights its role as a central metastore for Apache Spark, Flink, Trino, and BigQuery, supporting both Apache Iceberg and Hive formats. It also introduces the concept of 'Lakehouse runtime catalog federation,' allowing engines like BigQuery and Managed Spark to discover data residing in Snowflake, Databricks, and AWS Glue.
