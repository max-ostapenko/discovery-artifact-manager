# Iceberg support for timestamp precision autodetection

**Date:** 2026-06-05  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has extended timestamp precision autodetection support to Iceberg external tables and clarified Python UDF runtime options.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly supports the Iceberg External Table format. This allows developers to define how subsecond digits are handled (e.g., truncated or preserved as timestamp(6) or timestamp(12)) during schema autodetection for Iceberg. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to clarify that it specifically governs Python UDF container instances rather than generic Cloud Run instances.
