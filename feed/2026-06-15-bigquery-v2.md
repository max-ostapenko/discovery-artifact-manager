# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-15  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, simplifying access control debugging.

## Details

Descriptions for methods across datasets, jobs, tables, models, and routines now include a dedicated '# IAM Permissions' section. For example, the documentation for `tabledata.insertAll` now explicitly states it requires `bigquery.tables.updateData`, `bigquery.tables.get`, and `bigquery.datasets.get`. Additionally, `timestampTargetPrecision` fields in `ExternalDataConfiguration` and `JobConfigurationLoad` now formally document support for Iceberg External Tables.
