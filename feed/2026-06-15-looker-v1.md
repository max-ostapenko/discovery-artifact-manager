# Looker Core adds Soft-Delete Recovery and Release Channels

**Date:** 2026-06-15  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance undeletion and configurable release channels (Rapid, Regular, Stable). Developers also gain visibility into why instances were soft-deleted, such as billing issues or trial expirations.

## Details

A new `undelete` method has been added to the `instances` resource, paired with a `showDeleted` boolean parameter on the `list` method to help manage instance lifecycles. The `Instance` schema now includes `releaseChannel` for controlling update frequency and `acceleratedSecurityPatchEnabled` for opting into faster security updates. Additionally, new output-only fields `softDeleteReason` and `suspendedTime` provide diagnostic context for instances that have been suspended or deleted.
