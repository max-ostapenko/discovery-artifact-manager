# Looker Core adds Soft-Delete and Release Channels

**Date:** 2026-06-15  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and introduces configurable release channels for update management.

## Details

The Looker API has introduced a soft-delete lifecycle, allowing developers to recover instances using the new `projects.locations.instances.undelete` method. The `list` method now accepts a `showDeleted` boolean to track these instances. Furthermore, the `Instance` schema adds `releaseChannel` (RAPID, REGULAR, STABLE) and `acceleratedSecurityPatchEnabled`, giving admins more control over the update cadence and security posture of their Looker Core deployments. New output-only fields like `softDeleteReason` and `suspendedTime` provide additional visibility into why an instance was offlined.
