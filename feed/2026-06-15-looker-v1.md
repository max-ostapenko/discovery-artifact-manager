# Looker v1 adds Instance Undelete and Release Channels

**Date:** 2026-06-15  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports soft-deletion recovery and configurable release channels, allowing developers to restore deleted instances and control update cadences.

## Details

A new `undelete` method has been added to the instances resource, paired with a `showDeleted` boolean parameter on the `list` method to identify recoverable instances. The `Instance` schema now includes a `releaseChannel` field (supporting RAPID, REGULAR, and STABLE) and an `acceleratedSecurityPatchEnabled` toggle for maintenance control. Additionally, new output-only fields like `softDeleteReason` and `suspendedTime` provide visibility into why an instance was taken offline, distinguishing between billing issues, trial expirations, and manual deletions.
