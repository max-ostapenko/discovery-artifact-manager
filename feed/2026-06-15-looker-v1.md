# Looker Core adds Soft Delete recovery and Release Channels

**Date:** 2026-06-15  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 introduces instance recovery capabilities and granular control over update cadences via new release channels.

## Details

Developers can now recover deleted Looker instances using the new `instances.undelete` method. To support this, `instances.list` now includes a `showDeleted` query parameter, and the `Instance` schema has been expanded with `softDeleteReason` and `suspendedTime` fields. Additionally, a new `releaseChannel` field allows instances to be pinned to RAPID, REGULAR, or STABLE update tracks, and an `acceleratedSecurityPatchEnabled` flag has been added for faster security deployments.
