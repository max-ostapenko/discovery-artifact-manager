# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-16  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, simplifying access troubleshooting.

## Details

Methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now include an '# IAM Permissions' section in their descriptions. For example, jobs.get now clarifies that it requires bigquery.jobs.get or bigquery.jobs.create if the caller is the job creator. Additionally, timestampTargetPrecision in ExternalDataConfiguration and JobConfigurationLoad now explicitly supports Iceberg External Tables, and ExternalRuntimeOptions clarifies its use for Python UDF containers.
