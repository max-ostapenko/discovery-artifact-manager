# BigQuery API docs now explicitly list IAM permissions

**Date:** 2026-06-16  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, from dataset management to job execution.

## Details

Descriptions for methods across datasets, jobs, models, routines, tables, and rowAccessPolicies now explicitly state which IAM permissions (e.g., bigquery.datasets.get, bigquery.jobs.create) are required. Additionally, timestampTargetPrecision now supports Iceberg External Tables, and transferredBytes statistics have been clarified for BigQuery Omni cross-cloud queries to reflect data movement back to Google Cloud.
