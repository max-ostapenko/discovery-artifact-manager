# BigQuery API Docs Now Explicitly List IAM Requirements

**Date:** 2026-06-16  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

A massive documentation update for BigQuery v2 explicitly defines the IAM permissions required for nearly every method, from dataset management to job execution. It also adds Iceberg support for timestamp precision in external tables.

## Details

The update adds detailed IAM permission requirements to the descriptions of methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables. For instance, jobs.get now clarifies that it requires bigquery.jobs.get or bigquery.jobs.create if the caller is the job owner. Beyond documentation, ExternalDataConfiguration and JobConfigurationLoad now support timestampTargetPrecision for the Iceberg format, and ExternalRuntimeOptions clarifies concurrency settings for Python UDF containers.
