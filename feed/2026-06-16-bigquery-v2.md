# BigQuery API Docs Now Explicitly Detail IAM Requirements

**Date:** 2026-06-16  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, clarifying access control for datasets, jobs, and tables.

## Details

Method descriptions across the entire API surface—including datasets, jobs, models, routines, rowAccessPolicies, and tables—now include an explicit '# IAM Permissions' section. For instance, jobs.getQueryResults now specifies the need for bigquery.jobs.get and bigquery.tables.getData. Additionally, schema descriptions for ExternalDataConfiguration and JobConfigurationLoad were updated to include Iceberg External Table support for timestampTargetPrecision, and ExternalRuntimeOptions now references Python UDF containers instead of generic Cloud Run instances.
