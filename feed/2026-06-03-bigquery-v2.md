# BigQuery adds Iceberg support for timestamp precision

**Date:** 2026-06-03  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery's timestamp autodetection precision settings now officially support Iceberg External Tables, and UDF concurrency documentation has been clarified.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` has been updated to include support for the Iceberg External Table format. This allows developers to manage timestamp truncation (e.g., microsecond vs. nanosecond precision) during schema autodetection for Iceberg data. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was refined to clarify it applies specifically to Python UDF container instances.
