# Iceberg support for timestamp precision autodetection

**Date:** 2026-06-03  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded timestamp precision autodetection to include Iceberg External Tables and clarified concurrency settings for Python UDFs.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly supports the Iceberg External Table format. This allows developers to control whether auto-detected TIMESTAMP columns use 6 or 12 digits of subsecond precision when working with Iceberg. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to clarify its role in managing Python UDF container instances rather than generic Cloud Run instances.
