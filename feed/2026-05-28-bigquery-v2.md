# Iceberg support for timestamp autodetection

**Date:** 2026-05-28  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded timestamp autodetection precision settings to Iceberg External Tables and clarified concurrency settings for Python UDFs.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly supports the Iceberg External Table format alongside CSV, Parquet, and Avro. This allows developers to control sub-second precision (e.g., 6 or 12 digits) during autodetection for Iceberg sources. Additionally, the `containerRequestConcurrency` field in `ExternalRuntimeOptions` has been updated to clarify that it specifically controls concurrency for Python UDF container instances.
