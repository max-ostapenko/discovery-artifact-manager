# Looker adds Soft Delete recovery and Release Channels

**Date:** 2026-06-20  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance undeletion and explicit release channel selection (Rapid, Regular, Stable) for update management.

## Details

A new `instances.undelete` method has been introduced, allowing recovery of soft-deleted instances. To support this, the `instances.list` method now accepts a `showDeleted` boolean parameter. The `Instance` schema has been expanded with `releaseChannel` (supporting RAPID, REGULAR, and STABLE), `acceleratedSecurityPatchEnabled`, and output-only fields like `softDeleteReason` and `suspendedTime` to provide context on why an instance was deactivated (e.g., billing issues or trial expiration).
