# Looker Core adds Soft-Delete recovery and Release Channels

**Date:** 2026-06-20  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and provides granular control over update frequency with release channel settings.

## Details

The Looker v1 API has introduced a 'soft-delete' lifecycle for instances. A new `instances.undelete` method allows for the recovery of deleted instances, while the `instances.list` method now accepts a `showDeleted` boolean parameter. Additionally, the `Instance` schema now includes `releaseChannel` (supporting RAPID, REGULAR, and STABLE tracks) and `acceleratedSecurityPatchEnabled` for better maintenance management. New output-only fields `softDeleteReason` and `suspendedTime` provide visibility into why an instance was deactivated, whether due to billing issues, trial expiration, or customer request.
