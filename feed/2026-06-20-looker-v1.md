# Looker Core adds Soft Delete and Release Channels

**Date:** 2026-06-20  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and introduces configurable release channels for update management.

## Details

The Looker API has introduced a 'soft delete' lifecycle, allowing developers to list deleted instances using the new `showDeleted` parameter on `instances.list` and restore them via the new `instances.undelete` method. The `Instance` schema now includes `softDeleteReason` and `suspendedTime` to provide context on why an instance was removed. Additionally, a new `releaseChannel` field allows users to choose between RAPID, REGULAR, and STABLE update cadences, while `acceleratedSecurityPatchEnabled` offers more control over security maintenance.
