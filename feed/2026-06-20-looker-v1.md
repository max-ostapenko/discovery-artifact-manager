# Looker API adds Soft-Delete Recovery and Release Channels

**Date:** 2026-06-20  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides visibility into soft-deleted instances. Additionally, developers can now configure instance update cadences using the new releaseChannel field.

## Details

The instances.list method gains a showDeleted parameter, allowing discovery of instances in a soft-deleted state. A new instances.undelete method enables restoration of these instances, with metadata provided in the new softDeleteReason and suspendedTime fields. On the configuration side, the Instance schema now includes releaseChannel (RAPID, REGULAR, STABLE) for managing update frequency and acceleratedSecurityPatchEnabled for faster security rollouts.
