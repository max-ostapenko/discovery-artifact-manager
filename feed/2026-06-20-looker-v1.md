# Looker Core adds Soft Delete recovery and Release Channels

**Date:** 2026-06-20  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports instance recovery via a new undelete method and provides granular control over update cycles with release channel settings.

## Details

The API introduces a significant update to instance lifecycle management. Developers can now use the new `instances.undelete` method to restore instances that were soft-deleted due to billing issues, trial expirations, or manual requests. To support this, `instances.list` now accepts a `showDeleted` parameter, and the `Instance` schema includes new metadata fields like `softDeleteReason` and `suspendedTime`. Additionally, the API now supports `releaseChannel` selection (RAPID, REGULAR, STABLE) and an `acceleratedSecurityPatchEnabled` flag for more control over how and when instances are updated.
