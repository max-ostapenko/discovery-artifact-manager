# Iceberg support for timestamp precision autodetection

**Date:** 2026-05-29  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has expanded timestamp precision autodetection to Iceberg External Tables and clarified concurrency settings for Python UDFs.

## Details

The `timestampTargetPrecision` field in both `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly supports the Iceberg External Table format alongside CSV, Parquet, and AvRO. This allows for more granular control over how timestamp precisions (like microsecond vs nanosecond) are handled during schema autodetection. Additionally, the description for `containerRequestConcurrency` in `ExternalRuntimeOptions` was updated to specify its role in Python UDF container instances, providing better clarity for developers using remote functions.
