# Iceberg support for timestamp autodetection precision

**Date:** 2026-05-29  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery documentation now confirms support for Iceberg External Tables in timestamp autodetection precision settings and clarifies Python UDF concurrency options.

## Details

The `timestampTargetPrecision` field within `ExternalDataConfiguration` and `JobConfigurationLoad` has been updated to include 'Iceberg External Table' as a supported format for timestamp autodetection. This allows developers to control how sub-second precision is handled when loading or querying Iceberg data. Additionally, the description for `ExternalRuntimeOptions.containerRequestConcurrency` was updated to specify it applies to Python UDF container instances, providing better context for developers managing external runtime environments.
