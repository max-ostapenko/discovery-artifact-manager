# BigLake API rebranded to Lakehouse API

**Date:** 2026-06-22  
**API:** `biglake.v1`  
**Impact:** Low  

## Summary

The BigLake API description has been updated to reflect its new identity as the Lakehouse API, signaling broader support for multi-engine data sharing and catalog federation.

## Details

The service description has been overhauled to position the API as a 'Lakehouse API' rather than just a BigLake Metastore. The update highlights support for a wider range of engines including Apache Spark, Flink, and Trino, alongside BigQuery. It also mentions 'Lakehouse runtime catalog federation,' which enables discovery and analysis of data across Snowflake, Databricks, and AWS Glue. While this is currently a documentation-only change, it signals a significant expansion in the service's scope.
