# BigLake API rebranded to Lakehouse API

**Date:** 2026-06-22  
**API:** `biglake.v1`  
**Impact:** Low  

## Summary

The BigLake API has been officially rebranded to the Lakehouse API in documentation, signaling a shift toward a unified metastore for multi-engine data lakes.

## Details

The API description has been updated to reflect its new identity as the Lakehouse API. This change highlights broader support for engines like Apache Spark, Flink, and Trino alongside BigQuery. It also introduces the concept of 'Lakehouse runtime catalog federation,' which allows Iceberg-compatible engines to discover and analyze data across Snowflake, Databricks, and AWS Glue. Note that this update is currently limited to documentation and does not change existing resource paths or method signatures.
