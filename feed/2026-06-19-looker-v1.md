# Looker adds Soft Delete recovery and Release Channels

**Date:** 2026-06-19  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and provides granular control over update cadences with release channel settings.

## Details

The Looker v1 API has introduced a 'soft delete' lifecycle. A new `instances.undelete` method allows for the restoration of instances, while the `instances.list` method now supports a `showDeleted` parameter to view suspended resources. To support this, the `Instance` schema includes new fields: `softDeleteReason` (tracking billing issues, trial expirations, or manual requests), and `suspendedTime`. Additionally, developers can now manage update cadences via the `releaseChannel` field (RAPID, REGULAR, STABLE) and toggle `acceleratedSecurityPatchEnabled` for faster security updates.
