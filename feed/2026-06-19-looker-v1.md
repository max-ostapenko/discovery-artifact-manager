# Looker adds Soft Delete, Undelete, and Release Channels

**Date:** 2026-06-19  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker instances now support a soft-delete lifecycle, allowing for instance recovery and better visibility into why an instance was suspended.

## Details

The Looker v1 API has introduced a significant update to instance lifecycle management. A new `undelete` method has been added to the instances resource, paired with a `showDeleted` query parameter on the `list` method to help locate recoverable instances. The `Instance` schema now includes `softDeleteReason` (tracking if an instance was removed due to billing, trial expiration, or customer request) and `suspendedTime`. Additionally, developers can now configure a `releaseChannel` (RAPID, REGULAR, or STABLE) to control the update cadence and toggle `acceleratedSecurityPatchEnabled` for faster security updates.
