# Looker Core adds Instance Recovery and Release Channels

**Date:** 2026-06-19  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker v1 now supports soft-deletion recovery and configurable release channels for instances, giving developers more control over instance lifecycle and update cadences.

## Details

A new `undelete` method has been added to the instances resource, allowing for the restoration of soft-deleted instances. To support this, the `instances.list` method now accepts a `showDeleted` boolean parameter, and the `Instance` schema includes new output-only fields: `softDeleteReason` (identifying if deletion was due to billing, trial expiry, or user request) and `suspendedTime`. Additionally, developers can now configure the `releaseChannel` (RAPID, REGULAR, STABLE) and toggle `acceleratedSecurityPatchEnabled` to manage how and when their instances receive updates.
