# Soft-Delete Recovery and Release Channels for Looker

**Date:** 2026-06-19  
**API:** `looker.v1`  
**Impact:** Medium  

## Summary

Looker Core now supports instance recovery via a new undelete method and introduces configurable release channels for updates.

## Details

A new `undelete` method has been added to the `instances` resource, complemented by a `showDeleted` flag in the `list` method to help manage lifecycle transitions. The `Instance` schema now includes `softDeleteReason` (tracking billing issues, trial expirations, or manual requests) and `suspendedTime`. Additionally, developers can now specify a `releaseChannel` (RAPID, REGULAR, STABLE) and toggle `acceleratedSecurityPatchEnabled` for finer control over maintenance and security updates.
