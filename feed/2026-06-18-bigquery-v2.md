# BigQuery API adds explicit IAM permission documentation

**Date:** 2026-06-18  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 Discovery document has been updated to explicitly list the IAM permissions required for nearly every method, providing better clarity for access control and debugging.

## Details

Descriptions for core resources—including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables—now feature a dedicated '# IAM Permissions' section. This clarifies complex requirements, such as the specific permissions needed for row access policy updates or the distinction between job creators and project owners. Additionally, schema documentation was updated for Iceberg External Tables support in timestamp precision settings and clarified transferredBytes metrics for BigQuery Omni.
