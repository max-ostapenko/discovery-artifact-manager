# BigQuery API now explicitly lists required IAM permissions

**Date:** 2026-06-18  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, simplifying security configuration and debugging.

## Details

Descriptions for core resources—including datasets, jobs, tables, models, routines, and rowAccessPolicies—now feature a dedicated '# IAM Permissions' section. This clarifies complex logic, such as how `jobs.get` and `jobs.cancel` behave differently for job creators versus other users. Additionally, documentation for `timestampTargetPrecision` was updated to include support for Iceberg External Tables, and `ExternalRuntimeOptions` now specifically references Python UDF containers.
