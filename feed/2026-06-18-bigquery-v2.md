# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-18  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

BigQuery has updated its API discovery document to explicitly list the specific IAM permissions required for nearly every method, providing much-needed clarity for security auditing and service account configuration.

## Details

The documentation updates cover datasets, jobs, tables, models, routines, and rowAccessPolicies. Notable clarifications include the logic for job ownership: for methods like `jobs.get` and `jobs.cancel`, the job creator only needs `bigquery.jobs.create` at the project level, while other users require resource-level permissions. Streaming inserts via `tabledata.insertAll` now explicitly list requirements for `bigquery.tables.updateData`, `bigquery.tables.get`, and `bigquery.datasets.get`. Additionally, `timestampTargetPrecision` in load and external data configurations now officially supports Iceberg External Tables.
