# BigQuery API Docs Now Explicitly List IAM Requirements

**Date:** 2026-06-18  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

A massive documentation update across the BigQuery v2 API now explicitly details the IAM permissions required for nearly every method, from dataset management to job execution.

## Details

The discovery document has been updated to include specific IAM permission requirements for almost all methods. For example, `jobs.getQueryResults` now explicitly notes it requires `bigquery.jobs.get` and `bigquery.tables.getData`. `tabledata.insertAll` now lists three specific permissions across tables and datasets. Beyond IAM, `timestampTargetPrecision` in `ExternalDataConfiguration` and `JobConfigurationLoad` now includes support for Iceberg External Tables, and `ExternalRuntimeOptions` descriptions were clarified to reference Python UDF containers.
