# BigQuery API adds explicit IAM permission documentation

**Date:** 2026-06-18  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, clarifying the exact roles needed for datasets, jobs, and tables.

## Details

While no functional code changes were introduced, descriptions for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now include a dedicated '# IAM Permissions' section. This clarifies complex scenarios, such as how jobs.getQueryResults requires different permissions depending on whether the caller is the job creator. Additionally, timestampTargetPrecision now explicitly mentions support for Iceberg External Tables, and JobStatistics2.transferredBytes has been clarified for BigQuery Omni use cases.
