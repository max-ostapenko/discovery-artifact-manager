# BigQuery API Docs Now Explicitly Detail IAM Requirements

**Date:** 2026-06-12  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method. While no functional changes were made, this provides much-needed clarity for fine-grained access control and debugging permission errors.

## Details

Methods across datasets, jobs, models, routines, and tables now explicitly list the required IAM permissions in their descriptions (e.g., bigquery.datasets.get, bigquery.jobs.create). Notable clarifications include how jobs.get and jobs.cancel behave differently if the caller is the job creator versus an administrator. Additionally, timestampTargetPrecision documentation now includes support for Iceberg External Tables, and ExternalRuntimeOptions clarifies that containerRequestConcurrency applies to Python UDF container instances.
