# BigQuery API Now Explicitly Documents IAM Requirements

**Date:** 2026-06-12  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method. This provides critical clarity for developers configuring service accounts and debugging access issues.

## Details

Descriptions for resources including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now explicitly list the required IAM permissions (e.g., bigquery.jobs.create, bigquery.tables.getData). Notable clarifications include how jobs.list redacts information for non-creators and the specific permissions needed for complex SQL operations (DDL/DCL) in jobs.query. Additionally, timestampTargetPrecision now officially supports Iceberg External Tables, and ExternalRuntimeOptions has been updated to clarify Python UDF container concurrency.
