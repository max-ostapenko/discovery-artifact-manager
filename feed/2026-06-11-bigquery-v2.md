# BigQuery API adds explicit IAM requirements to method docs

**Date:** 2026-06-11  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been overhauled to explicitly list required IAM permissions for nearly every API method, from job management to row-level security.

## Details

Descriptions for resources across datasets, jobs, tables, models, and rowAccessPolicies now include a dedicated '# IAM Permissions' section. This clarifies complex logic, such as jobs.get requiring either bigquery.jobs.get or bigquery.jobs.create (if the caller is the job owner). Additionally, timestampTargetPrecision documentation was updated to include support for Iceberg External Tables, and ExternalRuntimeOptions now specifically references Python UDF containers instead of generic Cloud Run instances.
