# BigQuery API Docs Now Explicitly Detail IAM Permissions

**Date:** 2026-06-11  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, providing much-needed clarity for security and access troubleshooting.

## Details

This massive documentation update covers datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables. It clarifies complex logic, such as how jobs.get and jobs.cancel behave differently if the caller is the job creator (requiring project-level bigquery.jobs.create) versus a third party (requiring resource-level permissions). Additionally, timestampTargetPrecision documentation in ExternalDataConfiguration and JobConfigurationLoad now explicitly includes support for Iceberg External Tables, and ExternalRuntimeOptions clarifies its use for Python UDF containers.
