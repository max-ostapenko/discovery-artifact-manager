# BigQuery API now explicitly documents IAM requirements

**Date:** 2026-06-17  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, simplifying the process of configuring service accounts and debugging 403 errors.

## Details

Descriptions for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now include an explicit '# IAM Permissions' section. For example, tabledata.insertAll now specifies it requires bigquery.tables.updateData, bigquery.tables.get, and bigquery.datasets.get. Additionally, timestampTargetPrecision in JobConfigurationLoad and ExternalDataConfiguration now explicitly supports Iceberg External Tables, and ExternalRuntimeOptions clarifies its use for Python UDF container instances.
