# BigQuery API adds explicit IAM permission documentation

**Date:** 2026-06-17  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to explicitly list the IAM permissions required for nearly every API method, from dataset management to job execution.

## Details

Descriptions for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables now include a dedicated '# IAM Permissions' section. For example, jobs.getQueryResults now explicitly notes it requires bigquery.jobs.get and bigquery.tables.getData. Additionally, timestampTargetPrecision now supports Iceberg External Tables, and transferredBytes in job statistics has been clarified for BigQuery Omni use cases.
