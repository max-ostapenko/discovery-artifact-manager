# BigQuery API docs now explicitly list IAM requirements

**Date:** 2026-06-17  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

BigQuery has updated its API discovery document to include specific IAM permission requirements for nearly every method, alongside minor schema description clarifications.

## Details

The primary change is the addition of explicit IAM permission documentation across the entire API surface, including datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables. For instance, jobs.getQueryResults now explicitly documents the need for bigquery.jobs.get and bigquery.tables.getData. Additionally, ExternalDataConfiguration and JobConfigurationLoad now officially list Iceberg External Table support for timestampTargetPrecision, and JobStatistics2 clarifies transferredBytes usage for BigQuery Omni.
