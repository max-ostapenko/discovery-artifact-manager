# BigQuery API docs now explicitly list IAM requirements

**Date:** 2026-06-17  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method. Additionally, Iceberg External Tables now support timestamp precision autodetection.

## Details

Method descriptions for datasets, jobs, models, routines, and tables now explicitly state which IAM permissions (e.g., bigquery.jobs.create, bigquery.tables.getData) are required. This includes nuanced logic, such as how jobs.get permissions differ if the caller is the job creator. On the schema side, timestampTargetPrecision in ExternalDataConfiguration and JobConfigurationLoad now officially supports Iceberg External Tables, and ExternalRuntimeOptions clarifies its use for Python UDF container instances.
