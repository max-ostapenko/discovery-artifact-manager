# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-17  
**API:** `bigquery.v2`  
**Impact:** Medium  

## Summary

BigQuery has updated its discovery document to include specific IAM permission requirements for nearly all API methods, simplifying least-privilege configuration.

## Details

The update adds explicit IAM permission requirements to the descriptions of methods across datasets, jobs, models, routines, and tables. For instance, `jobs.getQueryResults` now clearly states it requires `bigquery.jobs.get` and `bigquery.tables.getData`. Beyond IAM, `ExternalDataConfiguration` now includes Iceberg External Tables in its timestamp precision autodetection documentation, and `ExternalRuntimeOptions` clarifies that `containerRequestConcurrency` applies specifically to Python UDF container instances.
