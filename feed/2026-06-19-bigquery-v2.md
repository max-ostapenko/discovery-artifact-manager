# BigQuery API clarifies IAM requirements across all resources

**Date:** 2026-06-19  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to explicitly list the IAM permissions required for nearly every method, from dataset management to job execution.

## Details

While no functional code changes were introduced, the API documentation now includes a dedicated '# IAM Permissions' section for methods across datasets, jobs, models, routines, rowAccessPolicies, tabledata, and tables. This clarifies specific requirements, such as the distinction between needing `bigquery.jobs.get` versus `bigquery.jobs.create` when retrieving job metadata. Additionally, schema descriptions were updated to include Iceberg External Table support for `timestampTargetPrecision` and to clarify that `transferredBytes` specifically tracks BigQuery Omni data movement back to Google Cloud.
