# BigQuery API Docs Now Explicitly List IAM Requirements

**Date:** 2026-06-19  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated with detailed IAM permission requirements for almost every method, clarifying access control for datasets, jobs, and row-level security.

## Details

While no functional changes were made to the API surface, method descriptions across datasets, jobs, models, routines, rowAccessPolicies, and tables now include a dedicated '# IAM Permissions' section. For example, jobs.getQueryResults now explicitly notes it requires bigquery.jobs.get and bigquery.tables.getData. Additionally, timestampTargetPrecision now mentions support for Iceberg External Tables, and JobStatistics2.transferredBytes has been clarified for BigQuery Omni data movement tracking.
