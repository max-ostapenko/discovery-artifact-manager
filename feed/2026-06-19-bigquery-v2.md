# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-19  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, from job creation to row-level security management.

## Details

While no functional code changes were introduced, the documentation for almost all resources (datasets, jobs, models, routines, tables, and rowAccessPolicies) now explicitly defines the required IAM permissions. For example, `jobs.getQueryResults` now details the specific permissions needed for both the job creator and other users. Additionally, `timestampTargetPrecision` in `ExternalDataConfiguration` and `JobConfigurationLoad` now explicitly mentions support for Iceberg External Tables, and `JobStatistics2.transferredBytes` has been clarified regarding BigQuery Omni cross-cloud data movement.
