# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-19  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, providing much-needed clarity for security auditing and troubleshooting.

## Details

While no functional changes were made to the API surface, the documentation for methods across datasets, jobs, tables, routines, and rowAccessPolicies now explicitly states the required bigquery.* permissions. For example, tabledata.insertAll now documents its requirement for bigquery.tables.updateData, bigquery.tables.get, and bigquery.datasets.get. Additionally, timestampTargetPrecision descriptions in ExternalDataConfiguration and JobConfigurationLoad were updated to include support for Iceberg External Tables, and ExternalRuntimeOptions clarified that containerRequestConcurrency applies to Python UDF containers.
