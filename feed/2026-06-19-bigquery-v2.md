# BigQuery API Docs Now Explicitly List IAM Permissions

**Date:** 2026-06-19  
**API:** `bigquery.v2`  
**Impact:** Low  

## Summary

The BigQuery v2 discovery document has been updated to include specific IAM permission requirements for nearly every method, providing much-needed clarity for troubleshooting access issues.

## Details

Descriptions for core resources—including datasets, jobs, models, routines, and tables—now explicitly state the required IAM permissions (e.g., bigquery.datasets.get, bigquery.jobs.create). Notable clarifications include the filtering logic for jobs.list and the specific data-level permissions needed for complex query jobs involving DDL or DCL. Additionally, timestampTargetPrecision now officially supports Iceberg External Tables, and ExternalRuntimeOptions descriptions were updated to specify Python UDF container instances.
